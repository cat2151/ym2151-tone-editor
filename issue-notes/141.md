# issue agentに実装させたとき、Windows版の考慮が漏れて、Windows版のビルドで落ちるPRが生成された。対策をいくつか試す #141
[issues #141](https://github.com/cat2151/ym2151-tone-editor/issues/141)

# 案、cargo checkのtargetに、x86_64-pc-windows-gnu を指定する
- WSLで成功したもの
- これをagentに実行させる
```sh
rustup target add x86_64-pc-windows-gnu

cargo check --all-targets --target x86_64-pc-windows-gnu

# 以下のような出力ならOK
#  >Finished `dev` profile [unoptimized + debuginfo] target(s) in 1.47s
```

# 検討中
- GitHub Actionsで、日次バッチとして 3:00 ごろに実行させ、NGならissueを自動起票させるのも手かも
  - 狙いは、agentが実装したものがWindows考慮漏れか？を、人間が朝起きて気付ける用
- agent指示書のpromptに、前述のrustup と cargo checkを実行させるのも手かも
  - 狙いは、agentが実装したものがWindows考慮漏れか？を、agentがPRする前に検知させて自律的に修正させる用
