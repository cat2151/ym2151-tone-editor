# issue 機能削除：エンベロープ折れ線グラフ表示機能を削除する #234
[issues #234](https://github.com/cat2151/ym2151-tone-editor/issues/234)

- これまでの課題
    - PR230は、issue 218の解決策になっていない
    - 一番重要なsixel実装、がまったく解決していない
        - 安易なフォールバック実装に逃げたままになっている
        - それも含め、命令違反が多い
        - 全体に、対応が低品質
        - エンベロープのアルゴリズムも間違いが多く、ゼロから作り直すほうが速いレベル
- 分析
    - PR230のsessionはこうなっていた
        - 現在のソースがこうなっている、という詳細な分析
        - sixelのエンベロープ折れ線グラフ、という命令に対して、
            - sixelの実装済みの波形グラフ、とごっちゃにしている
        - sixelを使えという命令に対して、sixelを使わないフォールバックをより改善しよう、
            - という低品質の生成をしている
        - 途中までそこそこ正解に近い生成も含んだ生成をしていたのに、
            - 実装の瞬間に、
            - 安易なフォールバックに逃げている
            - その際の言い訳もほとんどない
            - それまでの論理の流れを無視して、
            - 安易なフォールバックに逃げた結果となっている
        - 全体にムダな論理の流れが多い
        - 全体にuserへの命令違反が多い
            - sixelを使えという命令に対し、従わないなど
    - まとめると
        - なにが重要かをわかっていない
        - 命令違反
        - 実装の瞬間に安易なフォールバックに逃げており、論理が破綻している
        - 全体に理解度が低い
        - 全体に生成の品質が低い
    - 仮説
        - ソースが大きいためcoding agentが対応誤り
        - context window食いつぶし
- 対策案
    - エンベロープ折れ線グラフは破棄する
    - 別のリポジトリでライブラリとしてゼロから作り直すほうが、
        - トータルで速い、と考える
